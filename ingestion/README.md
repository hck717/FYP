# Ingestion Layer

The ingestion layer runs entirely offline via **Apache Airflow** and populates all three local databases (PostgreSQL, Qdrant, Neo4j) with fresh financial data daily. Agents never hit external APIs at query time — all data is pre-ingested.

---

## Architecture

```
External APIs
    │
    ├── EODHD All-In-One  ──►  Airflow DAG (daily 01:00 UTC)
    └── FMP Ultimate      ──►  Airflow DAG (daily 02:00 UTC)
                                    │
                         ┌──────────┼──────────┐
                         ▼          ▼          ▼
               load_postgres.py  load_neo4j.py  load_qdrant.py
                         │          │             │
                         ▼          ▼             ▼ (+ Ollama embed)
                   [PostgreSQL]  [Neo4j]       [Qdrant]
                  time-series    knowledge     semantic
                  fundamentals    graph        vectors
```

### Three Database Roles

| Database | Container | Port | What is stored |
|---|---|---|---|
| **PostgreSQL 15** | `fyp-postgres` | 5432 | OHLCV prices, ratios, analyst estimates, earnings, macro indicators, dividends, splits, treasury rates |
| **Neo4j 5.15** | `fyp-neo4j` | 7474 / 7687 | Company nodes (85+ properties each), sector/industry metadata |
| **Qdrant** | `fyp-qdrant` | 6333 | News article embeddings (768-dim, `nomic-embed-text`) for semantic RAG |

## DAGs

### DAG 1: `eodhd_complete_ingestion`

- **Schedule:** Daily 01:00 UTC (09:00 HKT)
- **Fan-out:** 3 agent personas × 5 tickers = 15 parallel task groups
- **Task graph per ticker:** `scrape → [load_postgres ∥ load_neo4j ∥ load_qdrant] → summary`

| Agent Persona | Endpoints Fetched | Destination |
|---|---|---|
| `business_analyst` | financial_news (100 articles), sentiment_trends, company_profile | Qdrant (news embeddings) + PostgreSQL (sentiment) + Neo4j (company node) |
| `quantitative_fundamental` | realtime_quote, historical_prices_eod, intraday 1m/5m/15m/1h, sma, ema, options, fundamentals_bulk | PostgreSQL |
| `financial_modeling` | weekly/monthly prices, dividends, splits, earnings, IPO calendar, bulk EOD, analyst estimates, macro events | PostgreSQL |

**Retry policy:**
- Default tasks: 2 retries, 5 min interval
- Qdrant tasks: 3 retries, 2 min interval (Ollama can be slow on first embed)

### DAG 2: `fmp_complete_ingestion`

- **Schedule:** Daily 02:00 UTC
- **Purpose:** Dual-path data verification — FMP data cross-validates EODHD figures
- **Key data:** Analyst estimates, DCF model inputs, treasury rates, market risk premium, earnings transcripts, insider trades (Form 4), institutional holdings (13F)

## Triggering DAGs

```bash
# Get the scheduler container (docker compose v2)
SCHED=$(docker compose ps -q airflow-scheduler)

# List all registered DAGs
docker exec -it $SCHED airflow dags list

# Unpause and trigger EODHD ingestion
docker exec -it $SCHED airflow dags unpause eodhd_complete_ingestion
docker exec -it $SCHED airflow dags trigger eodhd_complete_ingestion

# Trigger FMP ingestion
docker exec -it $SCHED airflow dags unpause fmp_complete_ingestion
docker exec -it $SCHED airflow dags trigger fmp_complete_ingestion

# Check DAG run status
docker exec -it $SCHED airflow dags list-runs -d eodhd_complete_ingestion

# View task list for a DAG
docker exec -it $SCHED airflow tasks list eodhd_complete_ingestion

# View logs for a specific task
docker exec -it $SCHED airflow tasks logs \
  eodhd_complete_ingestion \
  eodhd_load_qdrant_business_analyst_AAPL \
  2026-02-24

# Test a single task (debug — skips XCom; "DAG run not found" warning is expected and harmless)
docker exec -it $SCHED airflow tasks test \
  eodhd_complete_ingestion \
  eodhd_load_qdrant_business_analyst_AAPL \
  2026-02-24
```

**Re-embed all tickers (e.g. after Qdrant collection reset):**
```bash
for ticker in AAPL GOOGL MSFT NVDA TSLA; do
  docker exec -it $SCHED airflow tasks test \
    eodhd_complete_ingestion \
    eodhd_load_qdrant_business_analyst_${ticker} \
    2026-02-24
done
```

| `query_logs` | Agent query audit trail (Phase 3) |
| `citation_tracking` | Which chunks were cited per query (Phase 3 self-improvement) |

### Qdrant

```bash
# Dashboard: http://localhost:6333/dashboard

# Collection status via curl
curl -s http://localhost:6333/collections/financial_documents | python3 -c "
import sys, json
d = json.load(sys.stdin)['result']
print(f'Points:  {d[\"points_count\"]}')
print(f'Status:  {d[\"status\"]}')
"

# Delete collection (when rebuilding)
curl -X DELETE http://localhost:6333/collections/financial_documents

# Semantic search test
python3 - <<'EOF'
import requests
q = requests.post(
    "http://localhost:11434/api/embeddings",
    json={"model": "nomic-embed-text", "prompt": "Apple iPhone revenue growth 2025"}
).json()["embedding"]

hits = requests.post(
    "http://localhost:6333/collections/financial_documents/points/search",
    json={"vector": q, "limit": 5, "with_payload": True}
).json()["result"]

for h in hits:
    print(f"{h['score']:.4f} | {h['payload'].get('ticker_symbol')} | "
          f"{h['payload'].get('title', '')[:70]}")
EOF
```

**Note on `indexed_vectors_count: 0`:** Normal. Qdrant's HNSW index is only built above 20,000 KB. At ~2,390 vectors × 768 dims ≈ 7 MB, Qdrant uses brute-force full-scan which is 100% accurate and faster for small collections.

## ETL Loader Details

### `load_postgres.py`

- Auto-detects data type (timeseries vs fundamental) from `data_name` field
- Upserts to `raw_timeseries` or `raw_fundamentals` using MD5 hash deduplication
- Stores raw API response as `payload JSONB`
- Handles both EODHD and FMP payload structures

### `load_neo4j.py`

- Creates or updates `:Company` nodes via `MERGE ON ticker`
- Flattens nested EODHD company profile JSON into `_`-delimited property names (e.g. `Highlights_MarketCapitalization`)
- Stores 85+ financial properties per company node

### `load_qdrant.py`

- Cleans raw news text (strips HTML, normalises whitespace)
- Calls Ollama `/api/embed` with `nomic-embed-text` to produce 768-dim vectors
- Upserts to `financial_documents` collection with MD5 deduplication
- Includes Ollama warm-up probe with 3 retries before embedding begins
- Stores full payload: `ticker_symbol`, `agent_name`, `data_name`, `title`, `source`, `boost_factor`

### `ingest_neo4j_chunks.py`

- Proposition-based chunk ingestion for 10-K/10-Q filings
- Decomposes filing sections into atomic self-contained propositions
- Creates `:Chunk` nodes in Neo4j for graph-augmented retrieval (Phase 3)

## Daily Operations

The pipeline is fully automated:
```
Ollama running → Airflow scheduler triggers DAG at 01:00 UTC
    → scrape EODHD → load PostgreSQL + Neo4j + Qdrant (parallel)
    → FMP DAG at 02:00 UTC → load PostgreSQL
```

**Add a new ticker:**
```bash
# Edit .env
TRACKED_TICKERS=AAPL,GOOGL,MSFT,NVDA,TSLA,AMZN   # add AMZN

# Restart scheduler to pick up new env var
docker compose restart fyp-airflow-scheduler

# Trigger ingestion to backfill new ticker
docker exec -it $SCHED airflow dags trigger eodhd_complete_ingestion
```

**View scheduler logs:**
```bash
docker logs -f fyp-airflow-scheduler
```

## Troubleshooting

### Ollama connection failure (Qdrant embedding fails)

```bash
curl http://localhost:11434/api/tags   # should list models
ollama list                            # check nomic-embed-text is present
ollama pull nomic-embed-text           # re-pull if missing
ollama serve                           # restart if not responding
```

### Docker containers fail to start

```bash
docker compose logs --tail=50         # inspect errors
docker compose down -v                # full reset (deletes data volumes)
docker compose build --no-cache       # rebuild images
docker compose up -d
```

### PostgreSQL connection refused

```bash
docker ps | grep postgres             # check container is running
docker exec fyp-postgres pg_isready -U airflow
docker logs fyp-postgres --tail=20
```

### Neo4j connection refused

```bash
docker ps | grep neo4j
docker logs fyp-neo4j --tail=30
docker exec fyp-neo4j cypher-shell -u neo4j -p changeme_neo4j_password \
  "RETURN 'connected' AS status;"
```

### DAG not appearing in Airflow UI

```bash
# Check for syntax errors
docker exec $SCHED python /opt/airflow/dags/dag_eodhd_ingestion_unified.py

# Force DAG reload
docker exec $SCHED airflow dags reserialize
```

### Qdrant collection missing

```bash
# Rebuild by re-running all Qdrant tasks
SCHED=$(docker compose ps -q airflow-scheduler)
for ticker in AAPL GOOGL MSFT NVDA TSLA; do
  docker exec -it $SCHED airflow tasks test \
    eodhd_complete_ingestion \
    eodhd_load_qdrant_business_analyst_${ticker} \
    2026-02-24
done
```
